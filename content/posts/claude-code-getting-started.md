## Claude Code とは

Claude Code（クロード・コード）は、Anthropic が提供する **CLI 上で動作する AI コーディングアシスタント** です。ターミナルから `claude` と打つだけで、Claude と対話しながらコードを書いたり、ファイルを編集したり、テストを走らせたりできます。

似たようなツールに Cursor や GitHub Copilot がありますが、Claude Code の特徴は「**CLI からすべて完結する**」という点。エディタに縛られず、Bash の延長線として AI に作業を投げられるのが魅力です。

実際に使い倒してみて、「これは本当に日々の開発で便利」と感じた機能を 5 つに絞って紹介します。

## インストール

公式の手順どおり、npm で入れます。

```bash
npm install -g @anthropic-ai/claude-code
```

インストールが終わったら、プロジェクトのディレクトリで `claude` と打つだけで対話セッションが始まります。

```bash
cd my-project
claude
```

初回はブラウザで Anthropic アカウントのサインインを求められるので、案内どおり進めれば OK です。

## 最初のセッションでやること

Claude Code は「プロジェクトのことを何も知らない状態」から始まります。なので、最初に軽く現状を説明するのがコツ。

```
このプロジェクトは Next.js のブログサイトで、
記事は content/posts/ 配下に置いてあります。
```

これだけでプロジェクトの前提を理解してくれるので、次からの指示が格段にスムーズになります。

## 毎日使っている機能トップ 5

### 1. ファイルの読み書き

一番使います。「この関数を直して」と言うと、該当ファイルを読んで、問題を分析し、ファイルを修正してくれます。

ポイントは **ファイル名をできるだけ具体的に指定する** こと。「`src/utils/date.ts` の `formatDate` 関数を」のように伝えると、該当箇所への移動が一瞬です。

### 2. Bash の実行

テスト実行・ビルド確認・git 操作など、ターミナルで叩くコマンドをそのまま Claude に投げられます。

```
テストを走らせて、落ちているやつだけ直して
```

のような指示で、**テスト実行 → 失敗原因の分析 → 修正 → 再実行** までワンストップでやってくれます。

{{< notice info >}}
破壊的なコマンド（`rm -rf` や `git push --force` など）は、実行前に必ず確認プロンプトが出ます。勝手に消したりはしないので安心して任せられます。
{{< /notice >}}

### 3. CLAUDE.md で記憶を固定する

プロジェクトのルートに `CLAUDE.md` というファイルを置くと、Claude は毎セッション最初にこれを読み込みます。

私が書いている内容の例：

- プロジェクトの概要
- 使っているライブラリと前提
- よく使うコマンド
- 絶対に触らせたくないファイル（`.env` など）
- 独自のコーディング規約

毎回同じ説明を繰り返さなくていいので、作業のテンポが本当に変わります。個人的には **Claude Code の真価はここにある** と思っています。

### 4. TodoWrite でタスクを分解してくれる

長めの作業を頼むと、Claude が自動で「やることリスト」を作って、一つずつ進めてくれます。

例えば「ブログのテーマを完全に作り直して、BGM も追加したい」みたいな要望を投げると、

1. 現状の構造を確認する
2. 新しい HTML を書く
3. CSS を書き換える
4. BGM を追加する
5. ビルドして確認する
6. push する

のようにタスクを分解し、進捗を見せながら進めてくれます。途中で迷子にならずに済むので、長い作業ほど威力を発揮します。

### 5. スラッシュコマンド

セッション中に `/` から始まるコマンドが使えます。特によく使うのは：

- `/clear` — 会話をリセット。話が脱線したら即実行
- `/compact` — 会話を要約して再構成。長時間作業の途中で精度が落ちてきたときに
- `/help` — 使えるコマンド一覧
- `/agents` — サブエージェントの管理

`/clear` と `/compact` の使い分けは大事です。**完全に別の作業に移るときは `/clear`、今の作業は続けたいけどコンテキストが重いときは `/compact`**、という感じで使い分けています。

## ちょっとしたコツ

### エラーは「直して」でだいたい通る

ビルドエラーやテスト失敗の内容を貼り付けて「これ直して」と言うだけで、原因を分析して修正してくれます。長い説明を書く必要はありません。

### 権限は最初に allowlist に登録しておく

セッション中にファイル編集や Bash 実行の確認プロンプトが何度も出ます。よく使うコマンド（`git status`、`npm test` など）は事前に `settings.json` の allowlist に登録しておくと、ストレスなく作業できます。

### 重要な判断は人間が握る

Claude Code はかなり賢いですが、**アーキテクチャの方針や、本番に近い変更は必ず自分で確認する** 運用にしています。「AI に任せきり」ではなく、「AI を使って手早く選択肢を出してもらい、採否は自分で決める」くらいのバランスがちょうど良いです。

## まとめ

Claude Code は「ターミナル + AI」という、エンジニアのワークフローに自然に溶け込むツールです。

最初はコマンド一覧を覚えようとせず、**「やりたいことを普通の日本語で伝える」** ところから始めて OK。使いながら徐々に便利な機能を覚えていくのが近道です。

次回は、私が実際に使っている `CLAUDE.md` の中身を例に、プロジェクト設定のコツをまとめようと思います。
